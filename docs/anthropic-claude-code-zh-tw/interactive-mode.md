# 互動模式

> Claude Code 會話中鍵盤快捷鍵、輸入模式和互動功能的完整參考。

## 鍵盤快捷鍵

### 一般控制

| 快捷鍵           | 描述                                                        | 上下文                   |
| :------------ | :-------------------------------------------------------- | :-------------------- |
| `Ctrl+C`      | 取消當前輸入或生成                                                 | 標準中斷                  |
| `Ctrl+D`      | 退出 Claude Code 會話                                         | EOF 信號                |
| `Ctrl+L`      | 清除終端螢幕                                                    | 保留對話歷史                |
| `Ctrl+R`      | 反向搜尋命令歷史                                                  | 互動式搜尋先前的命令            |
| `上/下箭頭`       | 瀏覽命令歷史                                                    | 回想先前的輸入               |
| `Esc` + `Esc` | 倒回程式碼/對話                                                  | 將程式碼和/或對話恢復到先前的時間點    |
| `Tab`         | 切換[延伸思考](/zh-TW/docs/build-with-claude/extended-thinking) | 在思考開啟和思考關閉之間切換        |
| `Shift+Tab`   | 切換權限模式                                                    | 在自動接受模式、計劃模式和正常模式之間切換 |

### 多行輸入

| 方法       | 快捷鍵            | 上下文                    |
| :------- | :------------- | :--------------------- |
| 快速跳脫     | `\` + `Enter`  | 在所有終端中都有效              |
| macOS 預設 | `Option+Enter` | macOS 上的預設             |
| 終端設定     | `Shift+Enter`  | 在 `/terminal-setup` 之後 |
| 控制序列     | `Ctrl+J`       | 多行的換行字元                |
| 貼上模式     | 直接貼上           | 用於程式碼區塊、日誌             |

<Tip>
  在終端設定中配置您偏好的換行行為。執行 `/terminal-setup` 為 iTerm2 和 VS Code 終端安裝 Shift+Enter 綁定。
</Tip>

### 快速命令

| 快捷鍵     | 描述                    | 備註                                               |
| :------ | :-------------------- | :----------------------------------------------- |
| 開頭的 `#` | 記憶快捷鍵 - 新增到 CLAUDE.md | 提示檔案選擇                                           |
| 開頭的 `/` | 斜線命令                  | 參見[斜線命令](/zh-TW/docs/claude-code/slash-commands) |
| 開頭的 `!` | Bash 模式               | 直接執行命令並將執行輸出新增到會話中                               |

## Vim 編輯器模式

使用 `/vim` 命令啟用 vim 風格編輯，或透過 `/config` 永久配置。

### 模式切換

| 命令    | 動作           | 從模式    |
| :---- | :----------- | :----- |
| `Esc` | 進入 NORMAL 模式 | INSERT |
| `i`   | 在游標前插入       | NORMAL |
| `I`   | 在行首插入        | NORMAL |
| `a`   | 在游標後插入       | NORMAL |
| `A`   | 在行尾插入        | NORMAL |
| `o`   | 在下方開啟新行      | NORMAL |
| `O`   | 在上方開啟新行      | NORMAL |

### 導航（NORMAL 模式）

| 命令              | 動作         |
| :-------------- | :--------- |
| `h`/`j`/`k`/`l` | 向左/下/上/右移動 |
| `w`             | 下一個單字      |
| `e`             | 單字結尾       |
| `b`             | 上一個單字      |
| `0`             | 行首         |
| `$`             | 行尾         |
| `^`             | 第一個非空白字元   |
| `gg`            | 輸入開頭       |
| `G`             | 輸入結尾       |

### 編輯（NORMAL 模式）

| 命令             | 動作          |
| :------------- | :---------- |
| `x`            | 刪除字元        |
| `dd`           | 刪除行         |
| `D`            | 刪除到行尾       |
| `dw`/`de`/`db` | 刪除單字/到結尾/向後 |
| `cc`           | 變更行         |
| `C`            | 變更到行尾       |
| `cw`/`ce`/`cb` | 變更單字/到結尾/向後 |
| `.`            | 重複上次變更      |

## 命令歷史

Claude Code 維護當前會話的命令歷史：

* 歷史按工作目錄儲存
* 使用 `/clear` 命令清除
* 使用上/下箭頭導航（參見上面的鍵盤快捷鍵）
* **注意**：歷史展開（`!`）預設為停用

### 使用 Ctrl+R 進行反向搜尋

按 `Ctrl+R` 互動式搜尋您的命令歷史：

1. **開始搜尋**：按 `Ctrl+R` 啟動反向歷史搜尋
2. **輸入查詢**：輸入文字以在先前的命令中搜尋 - 搜尋詞將在匹配結果中被突出顯示
3. **導航匹配項**：再次按 `Ctrl+R` 循環瀏覽較舊的匹配項
4. **接受匹配項**：
   * 按 `Tab` 或 `Esc` 接受當前匹配項並繼續編輯
   * 按 `Enter` 接受並立即執行命令
5. **取消搜尋**：
   * 按 `Ctrl+C` 取消並恢復您的原始輸入
   * 在空搜尋上按 `Backspace` 取消

搜尋顯示匹配的命令，搜尋詞被突出顯示，使您能輕鬆找到並重複使用先前的輸入。

## 背景 bash 命令

Claude Code 支援在背景執行 bash 命令，讓您在長時間執行的程序執行時能繼續工作。

### 背景執行的運作方式

當 Claude Code 在背景執行命令時，它會非同步執行命令並立即返回背景任務 ID。Claude Code 可以在命令繼續在背景執行時回應新的提示。

要在背景執行命令，您可以：

* 提示 Claude Code 在背景執行命令
* 按 Ctrl+B 將常規 Bash 工具調用移到背景。（Tmux 使用者由於 tmux 的前綴鍵必須按兩次 Ctrl+B。）

**主要功能：**

* 輸出被緩衝，Claude 可以使用 BashOutput 工具檢索它
* 背景任務有唯一的 ID 用於追蹤和輸出檢索
* 當 Claude Code 退出時，背景任務會自動清理

**常見的背景命令：**

* 建置工具（webpack、vite、make）
* 套件管理器（npm、yarn、pnpm）
* 測試執行器（jest、pytest）
* 開發伺服器
* 長時間執行的程序（docker、terraform）

### 使用 `!` 前綴的 Bash 模式

透過在輸入前加上 `!` 前綴，直接執行 bash 命令而不經過 Claude：

```bash  theme={null}
! npm test
! git status
! ls -la
```

Bash 模式：

* 將命令及其輸出新增到對話上下文中
* 顯示即時進度和輸出
* 支援相同的 `Ctrl+B` 背景執行長時間執行的命令
* 不需要 Claude 解釋或批准命令

這對於在維護對話上下文的同時進行快速 shell 操作很有用。

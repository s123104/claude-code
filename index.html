快速安裝

完整支援 Linux/WSL 和 macOS 一鍵安裝腳本（start.sh），Windows 自動安裝腳本維修中，敬請期待！

支援 Linux、WSL、macOS，Windows 請稍後，將於修復後開放

Windows 用戶

# Windows 一鍵安裝腳本維修中，敬請期待！
setup.bat / setup.ps1 / start.bat
Windows 自動安裝腳本維修中，修復後將重新開放下載與一鍵安裝。

Linux/WSL 用戶

# 一鍵安裝 (推薦)
curl -fsSL https://raw.githubusercontent.com/s123104/claude-code/master/start.sh | bash
# 或下載後執行
curl -O https://raw.githubusercontent.com/s123104/claude-code/master/start.sh && chmod +x start.sh && ./start.sh
下載 start.sh

macOS 用戶

# 一鍵安裝 (推薦)
curl -fsSL https://raw.githubusercontent.com/s123104/claude-code/master/start.sh | bash
# 快速模式（跳過互動提示）
curl -fsSL https://raw.githubusercontent.com/s123104/claude-code/master/start.sh | bash -s -- --fast
下載 start.sh

工作流程與實戰範例

實際開發場景中的最佳實踐和工作流程範例

Sequential-Thinking 標準流程

  1. 需求分析：解析輸入、識別意圖、抽取技術需求、風險評估
  2. 計畫拆解：任務分解、依賴分析、旗標組合、執行順序規劃
  3. 逐步執行：前置檢查、指令執行、即時監控、錯誤處理
  4. 結果彙整：格式化輸出、後續建議、效能報告、學習記錄
# 典型指令範例
claude-code --create --template=nextjs --mcp
claude-code --scan --fix --lint --test
claude-code --build --deploy --monitor --security

MCP 多代理協作與自動化

  • 連接外部工具、資料庫、API，支援 OAuth 2.0、Postgres、JSON 設定
  • 可將 Claude Code 作為 MCP 伺服器，支援多種協議
  • 自訂斜線命令、管道整合、批次自動化
# MCP 整合範例
claude mcp add postgres-server
claude mcp add --connection-string "postgresql://..."

錯誤修復與自動診斷

  • 自動修復語法、相依、部署、效能等常見錯誤
  • 失敗時提供手動排查與建議
  • 支援多種測試框架和 CI/CD 整合
# 錯誤修復範例
claude-code --scan --verbose
claude-code --fix --auto
claude-code --test --coverage

Web UI 與遠端管理

  • 支援 PWA、行動裝置、Web Dashboard
  • 即時用量監控、Session 管理、REST API
  • 團隊協作和權限管理

常見問題與疑難排解

解決安裝、使用、和最佳化過程中的常見問題

安裝與啟動問題

  • Node.js 版本需 18 LTS，建議使用 nvm 管理
  • WSL 環境需 Windows 10 Build 19041+ 且啟用虛擬化
  • 避免使用 Windows 版本的 node/npm，應在 WSL 內重新安裝
  • npm 全域安裝目錄設置：~/.npm-global
  • 首次啟動需完成 API 金鑰設定

權限與安全問題

  • 所有敏感操作需明確批准，建議啟用 OpenTelemetry 監控
  • 預設僅開啟 View 權限，機密資訊用環境變數
  • 避免於生產環境使用 --dangerously-skip-permissions
  • 定期更新 Claude Code 以獲得最新安全修復

常見錯誤與排查

  • npm 配置污染：使用腳本自動清理 ~/.npmrc
  • node/npm 路徑錯誤：確保指向 WSL 而非 Windows
  • 虛擬化未啟用：BIOS 中開啟 VT-x/AMD-V
  • WSL 2 未設為預設：wsl --set-default-version 2
  • 磁碟空間不足：至少需要 1GB 可用空間

效能與最佳實踐

  • 大型專案建議分目錄分析，適當設定 --output-format、--timeout
  • 自動化批次審查、管道整合、結構化輸出
  • 定期 /compact，忽略大型目錄可降資源消耗
  • 使用 --cache 選項加速重複操作

🔧 故障排除

🆘 常見問題與解決方案

❌ Bash 版本過舊

問題：腳本提示 bash 版本低於 4.0

解決：腳本會自動檢測並提示升級，也可手動執行：

# macOS 用戶 brew install bash echo "$(brew --prefix)/bin/bash" | sudo tee -a /etc/shells chsh -s "$(brew --prefix)/bin/bash" # Linux 用戶 sudo apt update && sudo apt install -y bash # Ubuntu/Debian sudo yum update bash # CentOS/RHEL

❌ npm 權限錯誤

問題：npm 安裝時出現 EACCES 錯誤

解決：腳本會自動檢測並修復，或手動執行：

# 自動修復（推薦） ./start.sh --fast # 手動修復 mkdir -p ~/.npm-global npm config set prefix '~/.npm-global' echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc source ~/.bashrc

❌ Windows WSL 安裝失敗

解決步驟：

  1. 檢查 Windows 版本：需要 Build 19041+
  2. 啟用虛擬化：進入 BIOS 啟用 VT-x/AMD-V
  3. 管理員權限：確保以管理員身份執行 PowerShell
  4. 重新安裝：
wsl --unregister Ubuntu wsl --install -d Ubuntu-24.04

❌ Claude Code 無法執行

問題：安裝後 claude 指令找不到

解決：

# 檢查 PATH 設定 echo $PATH | grep -o '[^:]*npm[^:]*' # 重新安裝 npm uninstall -g @anthropic-ai/claude-code npm install -g @anthropic-ai/claude-code # 手動添加到 PATH export PATH="$HOME/.npm-global/bin:$PATH"

🔍 進階故障排除

📋 完整日誌分析

每次執行都會產生詳細日誌，可用於問題分析：

# 查看最新日誌 ls -la /tmp/claude_setup_*.log | tail -1 # 分析常見錯誤 grep -E "(ERROR|WARN)" /tmp/claude_setup_*.log # 檢查網路問題 grep -E "(download|curl|wget)" /tmp/claude_setup_*.log

🧹 完全重置安裝

如果遇到無法解決的問題，可以完全重置：

# 清理 npm 配置 rm -f ~/.npmrc npm config edit # 刪除所有內容並儲存 # 清理 NVM 安裝 rm -rf ~/.nvm # 重新執行安裝 ./start.sh --fast

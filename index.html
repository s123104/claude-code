快速安裝

官方安裝：跨平台以 npm 全域安裝；Windows 支援原生（無需 WSL）。

依官方文件建議：Node.js 18+，執行 npm install -g @anthropic-ai/claude-code

Windows 用戶

# Windows 原生安裝（無需 WSL）
winget install --id OpenJS.NodeJS.LTS -e --source winget
npm install -g @anthropic-ai/claude-code
claude --version
# 可選：Git Bash 路徑（部分功能更佳）
$env:CLAUDE_CODE_GIT_BASH_PATH = "C:\\Program Files\\Git\\bin\\bash.exe"

Linux/WSL 用戶

# 官方安裝（推薦）
npm install -g @anthropic-ai/claude-code
claude --version
# 若在 WSL 內 npm 誤判 OS
npm config set os linux
npm install -g @anthropic-ai/claude-code --force --no-os-check
環境特色
跨平台官方安裝（npm）：npm install -g @anthropic-ai/claude-code
健康檢查與診斷：claude doctor
WSL 安裝修復：npm config set os linux、--no-os-check
MCP 管理：claude mcp add / claude mcp list / add-from-claude-desktop
通知與偏好：claude config set --global preferredNotifChannel terminal_bell、/vim、/config

macOS 用戶

# 官方安裝（推薦）
brew install node
npm install -g @anthropic-ai/claude-code
# 驗證
claude --version

常見問題與疑難排解

解決安裝、使用、和最佳化過程中的常見問題

安裝與啟動問題

  • Node.js 版本需 18 LTS，建議使用 nvm 管理
  • WSL 環境需 Windows 10 Build 19041+ 且啟用虛擬化
  • 避免使用 Windows 版本的 node/npm，應在 WSL 內重新安裝
  • npm 全域安裝目錄設置：~/.npm-global
  • 首次啟動需完成 API 金鑰設定

權限與安全問題

  • 所有敏感操作需明確批准，建議啟用 OpenTelemetry 監控
  • 預設僅開啟 View 權限，機密資訊用環境變數
  • 避免於生產環境使用 --dangerously-skip-permissions
  • 定期更新 Claude Code 以獲得最新安全修復

常見錯誤與排查

  • npm 配置污染：使用腳本自動清理 ~/.npmrc
  • node/npm 路徑錯誤：確保指向 WSL 而非 Windows
  • 虛擬化未啟用：BIOS 中開啟 VT-x/AMD-V
  • WSL 2 未設為預設：wsl --set-default-version 2
  • 磁碟空間不足：至少需要 1GB 可用空間

效能與最佳實踐

  • 大型專案建議分目錄分析，適當設定 --output-format、--timeout
  • 自動化批次審查、管道整合、結構化輸出
  • 定期 /compact，忽略大型目錄可降資源消耗
  • 使用 --cache 選項加速重複操作
